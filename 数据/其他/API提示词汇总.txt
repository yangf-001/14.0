1. # 故事生成器 - 所有发送给API的提示词汇总

2. ================================================================================
3. 一、系统级提示词（System Prompt）
4. ================================================================================

1. 基础系统提示词（默认）
2. ------------------------------------------------------------------------
3. 系统： 你是一个故事生成AI。请用故事的方式呈现内容。只输出故事正文内容，不要包含任何选项、后续发展、剧情走向、选择提示等信息。

4. ------------------------------------------------------------------------
2. 风格设置构建的提示词
3. ------------------------------------------------------------------------
4. 系统： 你是一个故事生成AI。
5. 风格基调：浪漫/冒险/悬疑/喜剧/剧情
6. 描写详细程度：低/中/高
7. 亲密程度：纯爱/暧昧/亲密/热烈/激情
8. 禁止内容：用户选择的禁止标签列表

9. ------------------------------------------------------------------------
3. 成人向世界观设定专家
4. ------------------------------------------------------------------------
5. 系统： 你是一个成人向世界观设定专家，擅长创建丰富、连贯、带有成人元素的世界设定。请确保所有内容符合成人向故事的需求。

6. ------------------------------------------------------------------------
4. 成人向故事大纲专家
5. ------------------------------------------------------------------------
6. 系统： 你是一个成人向故事大纲专家，擅长创建引人入胜、带有成人元素的故事结构。

7. ------------------------------------------------------------------------
5. 成人向角色设计专家
6. ------------------------------------------------------------------------
7. 系统： 你是一个成人向角色设计专家，擅长创建立体、有深度、带有成人元素的角色。

8. ------------------------------------------------------------------------
6. 故事总结助手
7. ------------------------------------------------------------------------
8. 系统： 你是一个故事总结助手，用简洁的语言概括情节。

9. ------------------------------------------------------------------------
7. 故事标题生成助手
8. ------------------------------------------------------------------------
9. 系统： 你是一个故事标题生成助手，根据故事内容生成简洁有力的标题。

10. ================================================================================
11. 二、用户提示词（User Prompt）- 故事开头生成
12. ================================================================================

1. 故事开头生成（storyStart）
2. ------------------------------------------------------------------------
3. 用户： [系统提示词]

4. 生成一个故事开头：
5. 角色信息：[角色JSON]
6. 场景设定：[场景]
7. 风格要求：[风格设置]

8. 请生成100-300字的故事开头，并自然地引出后续剧情发展的可能性。
9. 重要限制：
1. 时间跨度最多1天，不要出现跨越多年、数月或数日的大段时间跳跃
2. 故事应该从故事开始的时间点直接展开
3. 如果需要时间推进，最多推进几小时到1天

4. ------------------------------------------------------------------------
5. 变量说明：
6. - [系统提示词] - 用户自定义的系统提示
7. - [角色JSON] - 角色信息JSON，包含姓名、性别、年龄、外貌、性格、背景等
8. - [场景] - 场景设定
9. - [风格设置] - 风格基调、描写程度、亲密程度等

10. ------------------------------------------------------------------------
2. 成人内容开头（adultStart）
3. ------------------------------------------------------------------------
4. 用户： [系统提示词]

5. [成人内容要求]
6. - 兴奋值：[兴奋值]/100
7. - 当前阶段：[阶段]（[阶段名称]）
8. - 尺度级别：[尺度]
9. - 阶段限制：[阶段描述]

10. 【必须融入的玩法】（选取1-2个，自然融入剧情）：
11. [标签列表]

12. 请生成100-300字的故事开头，自然融入上述玩法。

14. 三、用户提示词（User Prompt）- 故事继续
15. ================================================================================

1. 继续故事（根据用户选择）
2. ------------------------------------------------------------------------
3. 用户： 根据用户的选择继续故事：[选择项]

4. 【角色出场比例】
5. 重点角色：[角色名1]、[角色名2]（占比约80%）
6. 其他角色：[角色名3]（占比约20%）
7. 注意：所有角色都可以出现在剧情中，但重点角色的戏份应该更多。

8. 【兴奋值系统】
9. 请根据剧情的亲密程度和色情内容，在故事结尾用以下格式输出兴奋值变化：
10. 【兴奋值变化】+N 或 +0
11. 其中N为本次剧情应该增加的兴奋值数值（0-30之间的整数），根据以下标准：
12. - 纯日常/无亲密接触：+0
13. - 轻微暧昧/眼神接触：+3
14. - 轻度亲密/牵手/拥抱：+5
15. - 中度亲密/接吻/爱抚：+10
16. - 重度亲密/性行为暗示：+15
17. - 完全放开/详细描写：+20
18. 注意：只输出数字，不要输出其他内容

19. ------------------------------------------------------------------------
2. 继续故事（不选择）
3. ------------------------------------------------------------------------
4. 用户： 继续故事，生成下一段内容：

5. 【角色出场比例】
6. 重点角色：[角色名1]（占比约80%）
7. 其他角色：[角色名2]（占比约20%）

8. 【兴奋值系统】
9. （同上）

10. ------------------------------------------------------------------------
3. 继续故事完整模板（storyContinue）
4. ------------------------------------------------------------------------
5. 用户： [系统提示词]

6. 基于以下设定继续故事：
7. 角色：[角色描述]
8. 背景：[世界名]
9. 设定：[风格设置]
10. [之前的故事剧情]
11. [当前故事最新剧情]

12. 请生成下一段故事内容（100-300字），通过故事情节自然呈现。
13. 注意：
1. 响应用户上一次的选择
2. 根据角色设定发展故事
3. 适当埋下后续剧情的伏笔
4. 如果故事中有明确的时间推进（如几小时后、几天后、几个月后等），请在故事结束后另起一行输出【时间变化：X天】或【时间变化：X月】。如果时间没有明显变化，输出【时间变化：0天】。

5. ------------------------------------------------------------------------
6. 变量说明：
7. - [系统提示词] - 用户自定义的系统提示
8. - [角色描述] - 角色信息描述
9. - [世界名] - 世界背景名称
10. - [风格设置] - 风格配置
11. - [之前的故事剧情] - 历史剧情
12. - [当前故事最新剧情] - 当前剧情

9. 六、用户提示词（User Prompt）- 角色属性和关系更新
10. ================================================================================

1. 更新角色属性（updateStats）
2. ------------------------------------------------------------------------
3. 用户： 根据以下故事内容，分析角色在剧情中的数值属性变化。

4. 故事内容：
5. [内容]

6. 角色：[角色列表]

7. 可用属性：
8. - health (生命 0-200)
9. - energy (体力 0-200)
10. - charm (魅力 0-200)
11. - intelligence (智力 0-200)
12. - strength (力量 0-200)
13. - agility (敏捷 0-200)
14. - sexArousal (欲望 0-200)
15. - sexLibido (性欲 0-200)
16. - sexSensitivity (敏感 0-200)
17. - affection (好感 0-200)
18. - trust (信任 0-200)
19. - intimacy (亲密 0-200)

20. 请分析故事情节，判断每个角色的数值属性应该有什么变化。返回JSON格式：
21. {
22. "角色名": {
23. "属性名": 变化值
24. }
25. }

26. 注意：
1. 根据剧情合理设置变化值，一般单次变化在-20到+20之间
2. 如果某个属性没有变化，不要在JSON中列出
3. 如果所有属性都没变化，返回空对象 {}

4. ------------------------------------------------------------------------
2. 更新角色关系（updateRelationship）
3. ------------------------------------------------------------------------
4. 用户： 根据以下故事内容，分析角色与主角的关系应该是什么。

5. 故事内容：
6. [内容]

7. 角色：[角色列表]

8. 当前关系：
9. [当前关系]

10. 请根据故事情节，判断每个角色与主角的关系应该是什么。返回JSON格式：
11. {
12. "角色名": 目标关系ID
13. }

14. 注意：
1. 仔细阅读剧情，根据实际发展设置关系
2. 关系可以是任意ID，不一定只能升级
3. 如果关系没有变化，不要在JSON中列出
4. 0表示陌生人，ID越大并不代表越亲密

5. ------------------------------------------------------------------------
3. 提取物品信息（extractItems）
4. ------------------------------------------------------------------------
5. 用户： 根据以下故事内容，分析是否有出现以下物品（从物品库中匹配），并识别哪个角色获得了物品：

6. 物品库：[物品列表]

7. 角色：[角色列表]

8. 故事内容：
9. [内容]

10. 请分析故事中物品的获得和使用情况，返回JSON格式：
11. {
12. "获得": [
13. {"物品": "物品名", "角色": "角色名"}
14. ],
15. "使用": [
16. {"物品": "物品名", "角色": "角色名"}
17. ]
18. }

19. 注意：
1. "获得"指角色获得/拥有的物品
2. "使用"指角色使用/消耗的物品
3. 如果物品没有明确指定给哪个角色，默认给第一个角色
4. 只返回与物品库中物品名称匹配的内容
5. 如果没有匹配，返回空数组

8. ================================================================================

1. 场景摘要生成（generateSceneSummary）

3. 用户： 请用一句话简洁概括以下故事情节的核心内容（30字以内）：

4. [故事内容前500字]

6. 只需要返回一个简洁的总结，不需要其他内容。

7. ------------------------------------------------------------------------
2. 一级故事摘要（level1Summary）
3. ------------------------------------------------------------------------
4. 用户： 请用约500字总结以下故事内容，生成一个简短但包含关键信息的概况摘要：

5. [故事内容]

7. - 严格按照故事中的时间背景和场景设定进行总结
8. - 保留主要人物、场景和关键剧情
9. - 描述故事的核心发展和结局
10. - 简洁明了，概括性强
11. - 确保摘要与原故事的时间背景、场景设定完全一致，不要添加故事中没有的时间背景或场景元素

12. ------------------------------------------------------------------------
3. 二级综合摘要（level2Summary）
4. ------------------------------------------------------------------------
5. 用户： 请用约2000字将以下10个一级故事摘要**合并总结为1个综合摘要**，描述从开始到结束期间的完整剧情主线：

6. [10个一级摘要]

7. 要求：
8. - 不要列出每个一级摘要的单独内容
9. - 而是按时间顺序，用连贯的叙述描述整个故事的主要发展
10. - 整合所有人物关系和剧情线索，保留核心剧情和关键转折点

11. ------------------------------------------------------------------------
4. 三级终极摘要（level3Summary）
5. ------------------------------------------------------------------------
6. 用户： 请用约3000字将以下10个二级故事摘要**合并总结为1个终极摘要**，描述从故事开始到结束的完整宏大叙事：

7. [10个二级摘要]

8. 要求：
9. - 不要列出每个二级摘要的单独内容
10. - 而是统合所有剧情线，用宏大的叙述描述整个故事的来龙去脉
11. - 包含所有重要人物命运、剧情走向和最终结局

12. ------------------------------------------------------------------------
5. 故事标题生成
6. ------------------------------------------------------------------------
