# 故事生成器 - 所有发送给API的提示词汇总

================================================================================
一、系统级提示词（System Prompt）
================================================================================

1. 基础系统提示词（默认）
------------------------------------------------------------------------
系统： 你是一个故事生成AI。请用故事的方式呈现内容。只输出故事正文内容，不要包含任何选项、后续发展、剧情走向、选择提示等信息。

------------------------------------------------------------------------
2. 风格设置构建的提示词
------------------------------------------------------------------------
系统： 你是一个故事生成AI。
风格基调：浪漫/冒险/悬疑/喜剧/剧情
描写详细程度：低/中/高
亲密程度：纯爱/暧昧/亲密/热烈/激情
禁止内容：用户选择的禁止标签列表

------------------------------------------------------------------------
3. 成人向世界观设定专家
------------------------------------------------------------------------
系统： 你是一个成人向世界观设定专家，擅长创建丰富、连贯、带有成人元素的世界设定。请确保所有内容符合成人向故事的需求。

------------------------------------------------------------------------
4. 成人向故事大纲专家
------------------------------------------------------------------------
系统： 你是一个成人向故事大纲专家，擅长创建引人入胜、带有成人元素的故事结构。

------------------------------------------------------------------------
5. 成人向角色设计专家
------------------------------------------------------------------------
系统： 你是一个成人向角色设计专家，擅长创建立体、有深度、带有成人元素的角色。

------------------------------------------------------------------------
6. 故事总结助手
------------------------------------------------------------------------
系统： 你是一个故事总结助手，用简洁的语言概括情节。

------------------------------------------------------------------------
7. 故事标题生成助手
------------------------------------------------------------------------
系统： 你是一个故事标题生成助手，根据故事内容生成简洁有力的标题。

================================================================================
二、用户提示词（User Prompt）- 故事开头生成
================================================================================

1. 故事开头生成（storyStart）
------------------------------------------------------------------------
用户： [系统提示词]

生成一个故事开头：
角色信息：[角色JSON]
场景设定：[场景]
风格要求：[风格设置]

请生成100-300字的故事开头，并自然地引出后续剧情发展的可能性。
重要限制：
1. 时间跨度最多1天，不要出现跨越多年、数月或数日的大段时间跳跃
2. 故事应该从故事开始的时间点直接展开
3. 如果需要时间推进，最多推进几小时到1天

------------------------------------------------------------------------
变量说明：
- [系统提示词] - 用户自定义的系统提示
- [角色JSON] - 角色信息JSON，包含姓名、性别、年龄、外貌、性格、背景等
- [场景] - 场景设定
- [风格设置] - 风格基调、描写程度、亲密程度等

------------------------------------------------------------------------
2. 成人内容开头（adultStart）
------------------------------------------------------------------------
用户： [系统提示词]

[成人内容要求]
- 兴奋值：[兴奋值]/100
- 当前阶段：[阶段]（[阶段名称]）
- 尺度级别：[尺度]
- 阶段限制：[阶段描述]

【必须融入的玩法】（选取1-2个，自然融入剧情）：
[标签列表]

请生成100-300字的故事开头，自然融入上述玩法。

================================================================================
三、用户提示词（User Prompt）- 故事继续
================================================================================

1. 继续故事（根据用户选择）
------------------------------------------------------------------------
用户： 根据用户的选择继续故事：[选择项]

【角色出场比例】
重点角色：[角色名1]、[角色名2]（占比约80%）
其他角色：[角色名3]（占比约20%）
注意：所有角色都可以出现在剧情中，但重点角色的戏份应该更多。

【兴奋值系统】
请根据剧情的亲密程度和色情内容，在故事结尾用以下格式输出兴奋值变化：
【兴奋值变化】+N 或 +0
其中N为本次剧情应该增加的兴奋值数值（0-30之间的整数），根据以下标准：
- 纯日常/无亲密接触：+0
- 轻微暧昧/眼神接触：+3
- 轻度亲密/牵手/拥抱：+5
- 中度亲密/接吻/爱抚：+10
- 重度亲密/性行为暗示：+15
- 完全放开/详细描写：+20
注意：只输出数字，不要输出其他内容

------------------------------------------------------------------------
2. 继续故事（不选择）
------------------------------------------------------------------------
用户： 继续故事，生成下一段内容：

【角色出场比例】
重点角色：[角色名1]（占比约80%）
其他角色：[角色名2]（占比约20%）

【兴奋值系统】
（同上）

------------------------------------------------------------------------
3. 继续故事完整模板（storyContinue）
------------------------------------------------------------------------
用户： [系统提示词]

基于以下设定继续故事：
角色：[角色描述]
背景：[世界名]
设定：[风格设置]
[之前的故事剧情]
[当前故事最新剧情]

请生成下一段故事内容（100-300字），通过故事情节自然呈现。
注意：
1. 响应用户上一次的选择
2. 根据角色设定发展故事
3. 适当埋下后续剧情的伏笔
4. 如果故事中有明确的时间推进（如几小时后、几天后、几个月后等），请在故事结束后另起一行输出【时间变化：X天】或【时间变化：X月】。如果时间没有明显变化，输出【时间变化：0天】。

------------------------------------------------------------------------
变量说明：
- [系统提示词] - 用户自定义的系统提示
- [角色描述] - 角色信息描述
- [世界名] - 世界背景名称
- [风格设置] - 风格配置
- [之前的故事剧情] - 历史剧情
- [当前故事最新剧情] - 当前剧情

六、用户提示词（User Prompt）- 角色属性和关系更新
================================================================================

1. 更新角色属性（updateStats）
------------------------------------------------------------------------
用户： 根据以下故事内容，分析角色在剧情中的数值属性变化。

故事内容：
[内容]

角色：[角色列表]

可用属性：
- health (生命 0-200)
- energy (体力 0-200)
- charm (魅力 0-200)
- intelligence (智力 0-200)
- strength (力量 0-200)
- agility (敏捷 0-200)
- sexArousal (欲望 0-200)
- sexLibido (性欲 0-200)
- sexSensitivity (敏感 0-200)
- affection (好感 0-200)
- trust (信任 0-200)
- intimacy (亲密 0-200)

请分析故事情节，判断每个角色的数值属性应该有什么变化。返回JSON格式：
{
  "角色名": {
    "属性名": 变化值
  }
}

注意：
1. 根据剧情合理设置变化值，一般单次变化在-20到+20之间
2. 如果某个属性没有变化，不要在JSON中列出
3. 如果所有属性都没变化，返回空对象 {}

------------------------------------------------------------------------
2. 更新角色关系（updateRelationship）
------------------------------------------------------------------------
用户： 根据以下故事内容，分析角色与主角的关系应该是什么。

故事内容：
[内容]

角色：[角色列表]

当前关系：
[当前关系]

请根据故事情节，判断每个角色与主角的关系应该是什么。返回JSON格式：
{
  "角色名": 目标关系ID
}

注意：
1. 仔细阅读剧情，根据实际发展设置关系
2. 关系可以是任意ID，不一定只能升级
3. 如果关系没有变化，不要在JSON中列出
4. 0表示陌生人，ID越大并不代表越亲密

------------------------------------------------------------------------
3. 提取物品信息（extractItems）
------------------------------------------------------------------------
用户： 根据以下故事内容，分析是否有出现以下物品（从物品库中匹配），并识别哪个角色获得了物品：

物品库：[物品列表]

角色：[角色列表]

故事内容：
[内容]

请分析故事中物品的获得和使用情况，返回JSON格式：
{
  "获得": [
    {"物品": "物品名", "角色": "角色名"}
  ],
  "使用": [
    {"物品": "物品名", "角色": "角色名"}
  ]
}

注意：
1. "获得"指角色获得/拥有的物品
2. "使用"指角色使用/消耗的物品
3. 如果物品没有明确指定给哪个角色，默认给第一个角色
4. 只返回与物品库中物品名称匹配的内容
5. 如果没有匹配，返回空数组

七、用户提示词（User Prompt）- 故事摘要生成
================================================================================

1. 场景摘要生成（generateSceneSummary）
------------------------------------------------------------------------
用户： 请用一句话简洁概括以下故事情节的核心内容（30字以内）：

[故事内容前500字]

[用户选择了：选择项]

只需要返回一个简洁的总结，不需要其他内容。

------------------------------------------------------------------------
2. 一级故事摘要（level1Summary）
------------------------------------------------------------------------
用户： 请用约500字总结以下故事内容，生成一个简短但包含关键信息的概况摘要：

[故事内容]

要求：
- 严格按照故事中的时间背景和场景设定进行总结
- 保留主要人物、场景和关键剧情
- 描述故事的核心发展和结局
- 简洁明了，概括性强
- 确保摘要与原故事的时间背景、场景设定完全一致，不要添加故事中没有的时间背景或场景元素

------------------------------------------------------------------------
3. 二级综合摘要（level2Summary）
------------------------------------------------------------------------
用户： 请用约2000字将以下10个一级故事摘要**合并总结为1个综合摘要**，描述从开始到结束期间的完整剧情主线：

[10个一级摘要]

要求：
- 不要列出每个一级摘要的单独内容
- 而是按时间顺序，用连贯的叙述描述整个故事的主要发展
- 整合所有人物关系和剧情线索，保留核心剧情和关键转折点

------------------------------------------------------------------------
4. 三级终极摘要（level3Summary）
------------------------------------------------------------------------
用户： 请用约3000字将以下10个二级故事摘要**合并总结为1个终极摘要**，描述从故事开始到结束的完整宏大叙事：

[10个二级摘要]

要求：
- 不要列出每个二级摘要的单独内容
- 而是统合所有剧情线，用宏大的叙述描述整个故事的来龙去脉
- 包含所有重要人物命运、剧情走向和最终结局

------------------------------------------------------------------------
5. 故事标题生成
------------------------------------------------------------------------
用户： 请根据以下故事内容生成一个简短的故事标题（5-15个字）：

【故事开头】：
[故事开头前300字]

【故事结尾】：
[故事结尾前300字]

【出场角色】：[角色名1]、[角色名2]

要求：
1. 标题要能概括故事的核心内容或主题
2. 字数控制在5-15个字之间
3. 不要包含引号或其他符号
4. 直接返回标题，不要其他内容

九、亲密互动元素提示词
================================================================================

------------------------------------------------------------------------
用户： 使用[姿势名][姿势描述]，重点接触[身体部位]部位，进行[动作名]，对话内容：[对话内容]，以[风格名]风格进行，场景地点：[地点名]，角色分工：[角色名]，使用道具：[道具名]

================================================================================

------------------------------------------------------------------------

================================================================================
十一、兴奋值系统
================================================================================

1. 兴奋值变化输出
------------------------------------------------------------------------
输出格式：【兴奋值变化】+N

增加标准：
- 纯日常/无亲密接触：+0
- 轻微暧昧/眼神接触：+3
- 轻度亲密/牵手/拥抱：+5
- 中度亲密/接吻/爱抚：+10
- 重度亲密/性行为暗示：+15
- 完全放开/详细描写：+20

===========================================================================
注意：兴奋值范围0-100，当前阶段越高，允许的尺度越大。

================================================================================
十二、输出格式说明
================================================================================

1. 故事正文输出限制
------------------------------------------------------------------------
- 只输出故事正文内容
- 不要包含任何选项
- 不要包含后续发展提示
- 不要包含剧情走向提示
- 不要包含选择提示

2. JSON输出格式
------------------------------------------------------------------------
