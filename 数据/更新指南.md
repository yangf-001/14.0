# 项目更新指南

本指南将详细说明如何将新版本的AI故事生成器更新到GitHub仓库，并确保移动端也能同步更新。

## 一、更新到GitHub仓库步骤

### 1. 检查git状态

首先，确认当前git仓库的状态，查看哪些文件需要提交：

```bash
git status
```

### 2. 暂存所有更改

将所有修改的文件和新文件添加到暂存区：

```bash
git add .
```

### 3. 提交更改

提交所有暂存的更改，添加一个有意义的提交信息：

```bash
git commit -m "更新AI故事生成器到新版本"
```

### 4. 推送更改到GitHub

将本地提交推送到GitHub仓库：

```bash
git push origin master
```

## 二、确保移动端更新

### 1. 响应式设计检查

项目已经包含响应式设计的基本元素：

- `index.html` 中的 `<meta name="viewport" content="width=device-width, initial-scale=1.0">` 确保在移动设备上正确显示
- 项目包含 `mobile-nav` 元素，用于移动端导航

### 2. 移动端兼容性检查

在推送前，建议进行以下检查：

- 确保CSS中包含媒体查询，以适应不同屏幕尺寸
- 检查移动端导航功能是否正常工作
- 测试在移动设备上的触摸交互是否流畅

### 3. 移动端缓存处理

为确保移动端能够获取最新版本，建议：

- 在静态资源文件名中添加版本号或哈希值，避免浏览器缓存旧版本
- 确保服务器配置正确，不缓存关键文件

## 三、验证更新

### 1. 检查GitHub仓库

推送完成后，登录GitHub检查仓库是否已更新：

- 访问仓库页面，确认最新提交已显示
- 检查文件是否已正确上传

### 2. 测试移动端访问

在移动设备上测试应用：

- 清除浏览器缓存
- 访问应用URL
- 确认所有功能正常工作
- 验证UI在移动设备上显示正常

## 四、常见问题处理

### 1. 推送失败

如果推送失败，可能的原因：

- 网络连接问题：检查网络连接
- 权限问题：确保您有推送到仓库的权限
- 冲突问题：解决分支冲突后再推送

### 2. 移动端不更新

如果移动端没有显示最新版本：

- 清除浏览器缓存
- 强制刷新页面（iOS: 下拉刷新；Android: 长按刷新按钮）
- 检查服务器是否正确配置了缓存策略

## 五、最佳实践

1. **定期更新**：保持项目定期更新，避免一次性推送大量更改
2. **提交信息清晰**：使用有意义的提交信息，便于后续维护
3. **测试后再推送**：在本地充分测试后再推送到GitHub
4. **备份重要数据**：在更新前备份重要数据，以防出现问题
5. **分支管理**：对于重大更改，建议使用分支进行开发和测试

## 六、总结

通过以上步骤，您可以成功将新版本的AI故事生成器更新到GitHub仓库，并确保移动端能够同步更新。如果在更新过程中遇到任何问题，请参考常见问题处理部分或寻求技术支持。
